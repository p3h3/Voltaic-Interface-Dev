Set up VoltaicVCU to VESC CAN-Bus :

-Make sure you set the correct Baud Rate on all Devices!

-Do not confuse CANH and CANL!

-Have 60 Ohms - 240 Ohms between CANH and CANL (60 Ohm is best)!

-Make sure your CAN never exceeds 70% Data Capacity!

-Make sure you have the correct CAN-IDs set and saved in VESCs and VCU!

-If using just one VESC use VESC 1, with two use VESC 1(L) & VESC 2(R)

-Set VESC to CAN Mode "VESC"!

-Enable CAN Status 1,2,3,4,5 with at least 20Hz Data rate!

-Disable other Inputs in VESC like ADCs...

Acceleration Curve Modus :

-choose the Acceleration Behaviour!

1. Linear throttle

Standard fixed-rate limiter: changes at most rate_limit × Δt per tick Smooth, gradual ramp up/down toward target


2. Zero-cross aware

Slows down near or across zero for smoother reversals Instantly snaps to zero when backing off throttle


3. Fast linear

Same as Curve 1 but double the speed (2 × rate_limit) Quicker response with simple behavior


4. Instant response

No rate limiting at all Output jumps directly to target every tick


5. Linear with brake bypass

Limits upward changes, but allows instant drop when releasing throttle Responsive when backing off, smooth when accelerating


6. Accel/regen split + auto-zero

Separate limits for accel vs. braking Instantly zeros output when pedal is released


7. Same as Curve 5

Duplicate of Curve 5 for separate mode slot or config Same responsive-down, limited-up behavior

Voltaic Interface DBC User Manual:


In order to use CAN signals in a meaningful way, they must be converted into a human-readable format. This is achieved with the help of one or more DBC files that can be imported into the Voltaic Interface.


The values on the CAN bus are read according to the parameters of the DBC file. This means that they receive an offset, are scaled, the data type is taken into account and you tell the interface how many digits you expect so it can display the Value in the right format.


How to do that :


Based on experience, a DBC File has a hard life and sometimes does contain some intentional and unintentional errors, therefore we have left open the possibility to specify the parameters of the DBC file separately for the Voltaic Interface. This is done in the signal description of the individual signals; at any point in the description, a block with the following format can be inserted:


|Voltaic:[CONTENT]|


Where [CONTENT] should, of course, be replaced with your own content (without the "[ ]"). This content can contain one or more of the following parameters, each separated by a comma ( , ). The order is irrelevant:


name=[dataDistribution name for signal mapping]
type=[datatype]
factor=[factor to multiply with]
offset=[Offset (added after factor multiply)]
startBit=[Position of the first data bit of the signal]
unit=[unit shown in the gauges (max. 3 letters)]
displayName=[display Name inside the Gauges]
min=[minimal Value expected (example -99)]
max=[maximal Value expected (example 999)]
decPlaces=[decimal Places after the comma]

(the last 3 are important for the dynamic sizing of the values inside the gauges - default is 5 digits)
Example of a valid Block : |Voltaic:name=signal1,max=99,factor=2|


The parameters set in the “Voltaic Description” override the actual values of the DBC file in the Voltaic Interface. For example, if the signal is named “test333” and its description contains |Voltaic:name=test444|, the interface treats the signal as named “test444”. The same applies to all other parameters.

List of 100 pre-mapped Formula Student relevant Signals :


To simplify the Setup of a Vehicle we have predetermined 100 Values which are already pre-mapped to Tiles in our App.

To use these take the names from the list bellow and give the Signals in your DBC Files those names from below like this : " name=FS_vehicle_speed..."

We also specify in which unit we need those values below, so if your signals for instance is "W" and we demand "kW" just put factor=1000 and unit=kW in the DBC-Signal comment to create a kW signal which we can use further.
Also adjust the max and min values and the decimal Places after the "." accordingly.
